# Implementation Plan: FastAPI Skeleton (T02)

**Branch**: `005-t02-fastapi-skeleton` | **Date**: 2026-04-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/005-t02-fastapi-skeleton/spec.md`

## Summary

T02 is the first backend-code PR on top of the T01 tooling baseline. It lands four things, nothing more:

1. A runnable FastAPI application under `app/backend/` that starts without any secret or external dependency, exposes `GET /health` returning 200 JSON, and whose entrypoint matches the `uvicorn app.backend.main:app` target already hard-coded in the repo-root `Dockerfile`.
2. A committed OpenAPI contract at `app/backend/openapi.yaml` — the artefact that ADR-014 and constitution §14 require **before** any parallel sub-agent fan-out can start. A `generate_openapi` module regenerates it deterministically from the running app; a pytest test asserts zero drift between the committed YAML and the regenerated bytes.
3. A PII-safe structured logger (`structlog`-backed, JSON renderer) with a redaction processor that (a) masks values in a field allow-list (at minimum `candidate_email`) and (b) strips recognised email patterns from free-text `event` strings. This ships on day one so no later candidate-touching task has to retrofit the logging contract (constitution §15).
4. A minimal backend test layout under `app/backend/tests/` with three committed tests: the `/health` smoke, the PII-redaction test, and the OpenAPI-regeneration drift check. This establishes the test convention every downstream backend task copies.

No endpoints beyond `/health`. No database, no Alembic, no Vertex client, no auth middleware, no CORS. Those are T05, T04, and later tasks respectively; T02 is deliberately narrow so its diff is reviewable in one sitting.

## Technical Context

**Language/Version**: Python 3.12 — matches repo-root `pyproject.toml requires-python = ">=3.12,<3.13"` and the `python:3.12-slim-bookworm` base image in `Dockerfile`.
**Primary Dependencies** (runtime, added to `[project].dependencies` so Docker `uv sync --frozen --no-dev` picks them up):

- `fastapi` ≥ 0.115, < 0.120 — web framework + built-in OpenAPI schema export.
- `uvicorn[standard]` ≥ 0.32, < 0.36 — ASGI server. The `[standard]` extras pull in `httptools` + `uvloop` + `watchfiles`, matching the Dockerfile's single `uvicorn` CMD.
- `structlog` ≥ 24.4, < 25 — structured logger that supports a pipeline of processors (where the PII redactor lives).
- `pyyaml` ≥ 6.0, < 7 — deterministic YAML dump for `openapi.yaml`.

**Dev dependencies** (added to the existing `[dependency-groups].dev`):

- `pytest` ≥ 8.3, < 9 — test runner.
- `httpx` ≥ 0.27, < 0.29 — pinned to a version `fastapi.testclient.TestClient` is compatible with.

**Storage**: N/A — T02 does not touch any DB. Postgres arrives with T05.
**Testing**: pytest + `fastapi.testclient.TestClient` (which wraps `httpx`). Tests live under `app/backend/tests/` so mypy and ruff skip them where the config says so, and the `no-print-statements` pre-commit hook already excludes this path.
**Target Platform**: Linux container (`python:3.12-slim-bookworm`) for CI and prod; macOS / Linux dev machines for local `uvicorn --reload`.
**Project Type**: Monorepo web — backend slice only in this PR. Frontend (T03) is developed in parallel on a separate branch and consumes this PR's committed OpenAPI file.
**Performance Goals**:

- `GET /health` p99 < 50 ms on a single uvicorn worker running locally (no I/O, no DB).
- `uv run pytest app/backend/tests/` wall time < 30 s (SC-002).
- `uv run python -m app.backend.generate_openapi` wall time < 10 s (SC-003).

**Constraints**:

- **§15 PII** — zero raw candidate email values may appear in serialised log output. FR-010 test enforces.
- **§14 Contract-first** — `app/backend/openapi.yaml` must be committed **in this PR** (not a follow-up). T03, T04, T05, T06, T07, T09 all reference it.
- **§7 Docker parity** — the Dockerfile already `COPY app/backend ./app/backend` then `CMD ["uvicorn", "app.backend.main:app", ...]`. The runtime deps and entry point declared by T02 must match what that image expects; no dev-only paths.
- **§10 Forward-only migrations** — N/A at T02 (no migrations), but none of T02's scaffolding may make a later migration harder.
- **No secret at boot** (FR-003) — zero `os.environ[...]` required-key lookups in the T02 startup path. Env vars may be consulted for optional knobs (log level, log format) but must have safe defaults.
- **Pre-commit guardrails from T01** — `ruff --fix` + `ruff-format` + `gitleaks` + `detect-secrets` + `no-print-statements` + `no-direct-vertex-import` all pass on every T02-introduced file.

**Scale/Scope**: Single PR, ~12–15 new files, ~300 LOC of source + ~150 LOC of tests + the generated `openapi.yaml`. One committer (`backend-engineer` agent). No sub-agent fan-out from inside T02 — the fan-out T02 enables happens afterwards, on T03/T04/T06.

## Phase 0 — Outline & Research

Research output: [research.md](./research.md).

The spec has no `[NEEDS CLARIFICATION]` markers, but a handful of implementation-detail decisions sit below spec altitude and above `/speckit-tasks` altitude. Phase 0 resolves them with rationale rooted in an existing repo artefact so the reviewer can verify without external searches.

1. **Framework + ASGI server versions** — which FastAPI and uvicorn versions pin the runtime and match the Dockerfile's `uv sync --frozen --no-dev` expectation.
2. **OpenAPI serialisation** — how to get FastAPI's OpenAPI dict into a deterministic YAML file so the drift-check test actually detects drift (not whitespace churn).
3. **Drift-detection surface** — pre-commit hook vs. pytest test vs. CI-only check. The winner decides whether T10 (CI) needs to know about this yet.
4. **Logger choice and PII redaction mechanics** — `structlog` vs. stdlib `logging` + custom formatter; the exact processor shape; the field allow-list; the free-text email regex.
5. **Test framework** — pytest + `TestClient`; captured-log fixture; pytest layout inside `app/backend/tests/`.
6. **Health endpoint body shape** — what fields the JSON response includes; whether to bump a version on change.
7. **Boot without secrets** — which env vars the app reads at import/boot, and what their safe defaults are.

All seven decisions are resolved in `research.md` with a decision, a rationale, and rejected alternatives.

See [contracts/backend-contract.md](./contracts/backend-contract.md). A single consolidated document covers:
